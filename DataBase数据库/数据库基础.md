# 数据库基础



## 分类

- 关系型数据库

  Relational Database

- 非关系型数据库

  NoSQL Databse













## 关系型数据库

利用 关系模型（Relation Model）组织数据

数据存放于数据表（Table），

通过行（Column）和列（Row）展示

一个数据库中有多个数据表，通过关系相互关联起来



## CRUD

**CRUD**

- **C**reate ：**增加**数据
- **R**ead：**查询**数据
- **U**pdate：**更新**数据
- **D**elete：**删除**数据

> SLECT	?，？...	FROM	TableName；
>
> UPDATE	TableName	SET	列=?,列=？；
>
> DELETE FROM 	TableName；



### 构成

|               |                               |                            |
| :-----------: | :---------------------------: | :------------------------: |
|  **数据库**   |                               |      一些关联表的集合      |
|  **数据表**   |         **テーブル**          |                            |
|               |                               |                            |
|    **列**     |         **カラムと**          |    （一个人的全部数据）    |
| **行 / 记录** |         **レコード**          | （全部的姓名、全部的年龄） |
|   **字段**    |        **フィールド**         |      一个人的一个姓名      |
|               |                               |                            |
|   **主键**    | **主キー** / **プライマリキ** |     标记每一行的唯一性     |
|   **外键**    |         **外部キー**          |       两个表的共同列       |
|   **索引**    |       **インデックス**        |   方便查找表中记录（行）   |





ロール

という概念を使用し、データベース全体の権限を管理。









外键

可给一个表格设置多个外键，用来连接多个外部表格

表A：用户列表

表B：文章列表

通过外键（ForeignKey来关联这两个表，

例如在表B中添加表A中的主键，做为表B的外键user_id







**スキーマ**

模式，

> **スキーマ**は、
>
> データベースの構造を定義したものであり、テーブル、レコード、カラム、フィールド、主キー、インデックス、外部キーなどの設定を確認できる
>
> 複数のデータベースを管理し、1つのデータベースに複数のスキーマを権限ごとに管理できる。通常は意識せずにデフォルトで作成されるpublicスキーマを使用する

> ### 库、表、行、字段 的关系
>
> 传统型关系型数据库是以表格形式储存数据：
>
> - **数据库（databse）：**
>
>   每个项目都有对应的独立数据库
>
> - **数据表（table）：**
>
>   不同数据储存到不同表格中
>
> - **数据行（row）：**行
>
>   代表每一条具体数据
>
> - **字段（field）**：列
>
>   **每个表中储存的信息由字段决定**
>
> | 字段 | 字段 | 字段 |
